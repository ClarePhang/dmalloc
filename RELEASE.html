Debug Malloc (Dmalloc) Library Release Notes

Version: Release 5.2.2
Last-Version: 5.2.1
Date: September 8, 2003

Overview    New Functionality    Dmalloc Utility    settings.dist    Installation    Programming    Environmental Variables    Bugs Fixed

Overview

Bug fix release to fix a couple of problems. See the Bugs section for details.

New Functionality

None.

Dmalloc Utility Changes

None.

Changes to the settings.dist File

None.

Installation Changes

  • Added a section to the configure script to test for segfaults when getenv is called during the initialization process inside of malloc.

Programming Notes

  • Added a section to the manual to discuss usage of the library with the Cygwin environment.
  • Added a new ERROR_NOT_START_BLOCK (30) errno to report on addresses which are within an allocation but not pointing to the start of the block. In the past, the library would have returned ERROR_NOT_FOUND.

Environmental Variable Changes

None.

Bugs Fixed

  • Fixed a bug in the function argument pointer checking code. Dmalloc was improperly using the not-found error. Added to regression tests.
  • Improved the pointer checking code to properly check free space above an allocation. Added to regression tests.
  • Fixed a possible pthread lock leak in the dmalloc shutdown code under certain circumstances.
  • Added GETENV_SAFE configuration testing for problems when getenv is called during initialization of the library. This hopefully will better handle Cygwin environments.
