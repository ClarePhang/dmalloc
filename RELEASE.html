Debug Malloc (Dmalloc) Library Release Notes

Version: Release 4.8.0
Last-Version: 4.7.1
Date: November 10, 2000

Overview

General release to handle a couple of issues and introduce OpenBSD /usr/ports support.

New Functionality

  • Added OpenBSD port Makefile and other information. Hopefully this will work with Free, Net, ... BSD as well.

  • Added autoconf handling of #ident trying to fix problems with new egcs compiler.

    <rant>

    I really hate Unix's complete disregard of standards. I don't understand why the POSIX group or some other standards organization hasn't produced a program which must be compiled and run without errors for a system to be compliant. How hard is this? The program would methodically run through and test function call interfaces, system calls, handling of #ident and other directives, whatever. A published standard without an easy check means that every OS group can claim compliance but still miss by 5%.

    </rant>

Dmalloc Utility Changes

  • Made log-unknown the default behavior and depreciated the token.

  • Added log-known token to only report on known memory with file and line information.

Changes to the settings.dist File

    None.

Installation Changes

    None.

Programming Notes

  • The functions malloc_verify, dmalloc_verify now return [D]MALLOC_VERIFY_[NO]ERROR.

  • The function dmalloc_debug now takes an unsigned-integer flags (instead of a signed int) and it now returns the value of the old flags.

  • The function debug_current now returns an unsigned-integer flag value (instead of a signed int).

  • Just about every function now initializes the library. This was not the case before with some information routines.

Environmental Variable Changes

    None.

Bugs Fixed

  • Added autoconfiguration handling of #ident directives which broke on newer GCC versions. Idiots.

  • Fixed a bug with the divided-block free list in chunk.c found by Ian Badcoe. Thanks dude.
